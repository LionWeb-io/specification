[[evnt-AnnotationMovedAndReplacedInSameParent]]
===== Annotation moved in same parent and replaced existing annotation
<<existing-nodes, Existing node>> <<AnnotationMovedAndReplacedInSameParent.movedAnnotation>>
that was initially at index <<AnnotationMovedAndReplacedInSameParent.oldIndex>>
in <<AnnotationMovedAndReplacedInSameParent.parent>>'s annotations
was moved by <<AnnotationMovedAndReplacedInSameParent.indexOffset>> positions
to index (<<AnnotationMovedAndReplacedInSameParent.oldIndex>> + <<AnnotationMovedAndReplacedInSameParent.indexOffset>>),
replacing <<existing-nodes, current node>> <<AnnotationMovedAndReplacedInSameParent.replacedAnnotation>>{fn-org371}.
This deletes <<AnnotationMovedAndReplacedInSameParent.replacedAnnotation>>, and all its descendants (including annotation instances).
Does NOT change references to any of the deleted nodes.{fn-org285}

This conforms to <<movingAndReplacingNodes, the action of moving a node and replacing another one in one specific list>> — see that section for more information, diagrams, and examples.
