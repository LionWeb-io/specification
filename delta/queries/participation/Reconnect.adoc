[[qry-Reconnect]]
===== Reconnect (resume an existing participation)
The <<client>> has been technically disconnected, but still knows its <<participation-id>>.
Then the client can ask to reconnect to the repository.{fn-org349}

.Technical name
`Reconnect`

[horizontal]
.Request parameters
[[Reconnect.deltaProtocolVersion, `deltaProtocolVersion`]]deltaProtocolVersion:: <<deltaProtocolVersionType>>
MUST be the same id as used previously for the same <<Reconnect.participationId>>.{fn-org453}

[[Reconnect.clientId, `clientId`]]clientId:: <<clientIdType>>
Id of the reconnecting client.
MUST be the same id as used previously for the same <<Reconnect.participationId>>.{fn-org453}

[[Reconnect.repositoryId, `repositoryId`]]repositoryId:: <<repositoryIdType>>
Id of the repository to which the client wants to re-establish a participation.
MUST be the same id as used previously for the same <<Reconnect.participationId>>.{fn-org453}

[[Reconnect.participationId, `participationId`]]participationId:: <<participationIdType>>
The previously used <<participation-id>>.

[[Reconnect.lastReceivedSequenceNumber, `lastReceivedSequenceNumber`]]lastReceivedSequenceNumber:: <<eventSequenceType>>
Last <<event-sequence-number>> received by the client.

[[Reconnect.requestQueryId, `queryId`]]queryId:: <<queryIdType>>
Id of this query.

[[Reconnect.requestAdditionalInfos, `additionalInfos`]]additionalInfos:: <<additionalInfosType>>

[horizontal]
.Response parameters
[[Reconnect.lastSentSequenceNumber, `lastSentSequenceNumber`]]lastSentSequenceNumber:: <<eventSequenceType>>
Last <<event-sequence-number>> sent by the repository.

[[Reconnect.responseQueryId, `queryId`]]queryId:: <<queryIdType>>
Id of this query.

[[Reconnect.responseAdditionalInfos, `additionalInfos`]]additionalInfos:: <<additionalInfosType>>

.Errors
* <<err-invalidParticipation>> If the participation is not valid, or the participation doesn't match with <<Reconnect.deltaProtocolVersion>> and/or <<Reconnect.clientId>> and/or <<Reconnect.repositoryId>>.{fn-org354}
